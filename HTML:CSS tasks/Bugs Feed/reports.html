Warning “this field is required” isn’t displayed below the “Other” mandatory input after trying to save empty feedback.
Description: Warning “this field is required” isn’t displayed below the “Other” mandatory input after trying to save empty feedback. Notice: this bug can be reproduced on all feedback templates with the “Other” mandatory input included (“1 Month Teach Lead Feedback” and/or “PR: Manager & Teach Lead Feedback (Designer)”).
Preconditions:
  • Any opened (not closed or canceled) event (e.g., for Sandy Adams) is created;
  • Feedback requests (“1 Month Teach Lead Feedback” and/or “PR: Manager & Teach Lead Feedback (Designer)”) are sent.
  • The user is logged into the system https://impulse-dev.mev.cloud/ as an employee requested for leaving feedback (e.g., User Admin (admin@mev.com / pass)).
Steps to reproduce:
  1. Click on the “Calendar” in the left sidebar; Choose the event (see preconditions) in the list and click on its link;
  2. Click on the “Feedback” tab;
  3. Click on the “+Add” button of any related to the user feedback;
  4. Scroll down and click on the “Save” button.
  5. Pay attention to the required fields (marked with asterisk ‘*’).
Actual result: Warning “this field is required” isn’t displayed below the “Other” mandatory input.
Expected result: Warning “this field is required” is displayed below all mandatory fields (marked with asterisk ‘*’).
